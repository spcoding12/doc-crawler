# 📖 Doc Crawler：它是如何工作的？（通俗版）

如果你不懂编程，没关系！你可以把 **Doc Crawler** 想象成一个专门去数字图书馆“抄书”的聪明机器人。

---

## 🏗️ 那些文档网站是怎么设计的？

想象一下，你走进一个非常大的图书馆，里面有很多不同的书格。虽然书的内容不同，但摆放方式通常只有几种固定的“装修风格”：

1.  **装修风格（框架/Framework）**：
    -   有的图书馆像 **IKEA（Docusaurus）**：左边是目录，中间是正文，右上角是小标题。
    -   有的像 **MUJI（MkDocs）**：极简风格，侧边栏折叠得整整齐齐。
    -   **本质**：这些网站并不是手动一页页画出来的，而是用一种“模具”生成的。只要找到了模具的规律，我们就能一次性把所有书都搬走。

2.  **隐藏的地图（侧边栏/Navigation）**：
    -   大多数文档网站都有一个“侧边栏”，就像是书的**详细目录**。这个目录告诉机器人：哪一页在哪，它们之间是什么父子关系。

---

## 🤖 我们的机器人是怎么“搬”内容的？

Doc Crawler 搬书的过程可以简单分为三步：

### 第一步：找地图（导航提取）
机器人进站后，第一件事是寻找侧边栏。
-   **聪明模式**：它会先看一眼：“哦，这是 IKEA 风格的图书馆，我知道目录在左边那个特定的盒子里。”
-   **直觉模式（兜底）**：如果它没认出装修风格，它就会扫描页面上所有的链接。如果有一串链接长得很像，而且都在同一个域名下，它就会猜：“这大概就是我要找的目录了！”

### 第二步：智能阅读（内容提取）
找到具体的某一页后，机器人需要把“正文”抄下来，但它得避开周围的广告、搜索框、登录按钮。
-   **阅读模式**：就像手机浏览器的“阅读模式”一样，它会给页面上所有文字区域打分。文字越多、段落越长的地方，得分最高，它就认为那是“正文”。
-   **图片快照**：看到图片时，它会顺手把图片也下载下来，并贴在它抄好的 Markdown 笔记里。

### 第三步：深度大扫除（清理引擎）
这是最关键的一步。抄下来的草稿里经常会有垃圾，机器人会像拿着橡皮擦一样进行清理：
-   **擦除垃圾信息**：去掉那些没用的按钮（比如“点我复制内容”、“这一页有帮助吗？”）。
-   **修补乱码**：有时候文字搬运过程中会变样（比如把 `'` 变成了 `â€™`），它会自动把它们修正回来。
-   **整理标题**：它会确保每一页只有一个大标题，把重复的划掉，让笔记看起来像一本真正的电子书。

---

## 🎯 最终你得到的是什么？

最后，机器人会把所有抄好的笔记整理成两种样子给你：
1.  **合订本**：把几百页文档合并成一个很长的 Markdown 文件，适合直接喂给对话机器人（比如 ChatGPT、NotebookLM）让它学习。
2.  **文件夹版**：保留原来的目录结构，每个页面一个文件，方便你在本地自己阅读。

---

**总结**：Doc Crawler 就是一个能自动识别网站“长相”，绕过干扰项，把干货内容完整、干净地搬回家的自动化小秘书。
